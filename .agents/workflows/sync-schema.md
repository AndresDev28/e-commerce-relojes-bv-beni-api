# Antigravity Skill: Sync-Model-To-Frontend

## 1. Descripción
- **Propósito:** Automatizar la paridad de tipos entre los modelos de Sequelize (Back) y las interfaces de datos (Front).
- **Trigger:** Cambios detectados en cualquier archivo dentro de `database/models/*.js`.

## 2. Lógica de Ejecución (Protocolo)
1. **Extracción:** El agente lee el archivo del modelo en el Backend y extrae:
   - Nombre de la tabla.
   - Atributos (ej: `name`, `price`, `stock`).
   - Tipos de datos (STRING -> string, INTEGER -> number, etc.).
   - Nullability (`allowNull`).
2. **Transformación:** Convierte el esquema de Sequelize a un bloque de código TypeScript Interface.
3. **Proyección:** Busca en el repositorio hermano (`e-commerce-relojes-bv-beni`) la carpeta `src/types/` o `src/interfaces/`.
4. **Actualización:** Crea o actualiza el archivo `.ts` correspondiente con el nuevo esquema.

## 3. Reglas de la Skill
- **Seguridad:** No se deben exportar campos marcados como `private` o sensibles (ej: `password_hash`) hacia el frontend, respetando la **Global Rule #2**.
- **Consistencia:** El nombre de la interfaz en el Front debe ser el nombre del Modelo + `Interface` (ej: `WatchInterface`).
- **Traceability:** La Skill debe insertar un comentario en la cabecera del archivo generado: `// Auto-generated by Antigravity Sync-Skill - TraceID: {trace_id}`.
